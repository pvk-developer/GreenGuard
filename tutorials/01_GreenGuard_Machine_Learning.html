GreenGuard Machine LearningÂ¶

In this tutorial we will show you how to use GreenGuard to solve a Machine Learning problem defined via a Target Times table.

During the next steps we will:

  • Load demo target times and readings

  • Find available pipelines and load two of them as templates

  • Use GreenGuard AutoML to select the best template and hyperparameters for our problem

  • Build and fit a Machine Learning pipeline based on the found template and hyperparameters

  • Make predictions using the fitted pipeline

  • Evaluate how good the predictions are

0. Setup the loggingÂ¶

This step sets up logging in our environment to increase our visibility over the steps that GreenGuard performs.

[13]:

import logging;

logging.basicConfig(level=logging.INFO)
logging.getLogger().setLevel(level=logging.INFO)

import warnings
warnings.simplefilter("ignore")

1. Load the DataÂ¶

The first step is to load the data that we are going to use.

In order to use the demo data included in GreenGuard, the greenguard.demo.load_demo function can be used.

[14]:

from greenguard.demo import load_demo

target_times, readings = load_demo()

This will download some demo data from GreenGuard S3 demo Bucket and load it as the necessary target_times and readings tables.

The exact format of these tables is described in the GreenGuard README and docs:

Load your own DatasetÂ¶

Alternatively, if you want to load your own dataset, all you have to do is load the target_times and readings tables as pandas.DataFrame objects.

Make sure to parse the corresponding datetime fields!

import pandas as pd

target_times = pd.read_csv('path/to/your/target_times.csv', parse_dates=['cutoff_time'])
readings = pd.read_csv('path/to/your/readings.csv', parse_dates=['timestamp'])

2. Split the dataÂ¶

Once we have loaded the target_times and before proceeding to training any Machine Learning Pipeline, we will have split them in 2 partitions for training and testing.

In this case, we will split them using the train_test_split function from scikit-learn, but it can be done with any other suitable tool.

[21]:

from sklearn.model_selection import train_test_split

train, test = train_test_split(target_times, test_size=0.25, random_state=0)

3. Finding the available TemplatesÂ¶

The next step will be to select a collection of templates from the ones available in GreenGuard.

For this, we can use the greenguard.get_pipelines function, which will return us the list of all the available MLBlocks pipelines found in the GreenGuard system.

[22]:

from greenguard import get_pipelines

get_pipelines()

[22]:

['normalize_dfs_xgb_classifier',
 'unstack_normalize_dfs_xgb_classifier',
 'unstack_dfs_xgb_classifier',
 'unstack_lstm_timeseries_classifier',
 'unstack_double_lstm_timeseries_classifier']

Optionally, we can pass a string to select the pipelines that contain it:

[23]:

get_pipelines('dfs')

[23]:

['normalize_dfs_xgb_classifier',
 'unstack_normalize_dfs_xgb_classifier',
 'unstack_dfs_xgb_classifier']

Additionally, we can pass the keyword path=True to obtain a dictionary containing also the path to the pipelines instead of only the list of names.

[24]:

get_pipelines('dfs', path=True)

[24]:

 'unstack_dfs_xgb_classifier': '/home/usuario/Projects/GreenGuard/greenguard/pipelines/unstack_dfs_xgb_classifier.json'}

For the rest of this tutorial, we will select and use the templates unstack_normalize_dfs_xgb_classifier and normalize_dfs_xgb_classifier.

The unstack_normalize_dfs_xgb_classifier template contains the following steps:

  • Resample the data using a 10 minute average aggregation

  • Unstack the data by signal, so each signal is in a different column

  • Normalize the Turbine IDs into a new table to assist DFS aggregations

  • Use DFS on the readings based on the target_times cutoff times using a 1d window size

  • Apply an XGBoost Classifier

And the normalize_dfs_xgb_classifier template contains the above steps but without unstacking the data by signal.

[25]:

templates = [
    'unstack_normalize_dfs_xgb_classifier',
    'normalize_dfs_xgb_classifier'
]

4. Finding the best PipelineÂ¶

Once we have loaded the data, we create a GreenGuardPipeline instance by passing:

  • templates (string or list): the name of a template, the path to a template json file or a list that can combine both of them.

  • metric (string or function): The name of the metric to use or a metric function to use.

  • cost (bool): Whether the metric is a cost function to be minimized or a score to be maximized.

Optionally, we can also pass defails about the cross validation configuration:

  • stratify

  • cv_splits

  • shuffle

  • random_state

[26]:

from greenguard.pipeline import GreenGuardPipeline

pipeline = GreenGuardPipeline(templates, metric='f1', cv_splits=3)

Once we have created the pipeline, we can find which template and which combination of hyperparameters works best for our data by calling the tune method of our pipeline, passing its target_times and readings variables. This method will return a BTBSession session that will: - Select and tune templates. - If a template or hyperparameters that get a higher score than the previous one is found, automatically update our pipeline so that it uses that template with those hyperparameters. - Remove templates that donâ€™t work with the given data and focus on tuning only the ones that do.

[27]:

session = pipeline.tune(target_times, readings)

Once we have our session we can call itâ€™s method run with the amount of tuning iterations that we want to perform:

7. Save and load the pipelineÂ¶

Since the tuning and fitting process takes time to execute and requires a lot of data, you will probably want to save a fitted instance and load it later to analyze new signals instead of fitting pipelines over and over again.

This can be done by using the save and load methods from the GreenGuardPipeline.

In order to save an instance, call its save method passing it the path and filename where the model should be saved.

[39]:

path = 'my_pipeline.pkl'

pipeline.save(path)

Once the pipeline is saved, it can be loaded back as a new GreenGuardPipeline by using the GreenGuardPipeline.load method:

[40]:

new_pipeline = GreenGuardPipeline.load(path)

Once loaded, it can be directly used to make predictions on new data.

[41]:

predictions = new_pipeline.predict(test, readings)
